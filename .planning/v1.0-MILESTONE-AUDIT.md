### ⚠️ Missing Connection (Tech Debt)

**`logOutcomes()` not scheduled in `worker.ts`**

- `backend/src/collectors/outcome-logger.ts` exports `logOutcomes()` and works as a CLI (`npx tsx backend/src/collectors/outcome-logger.ts`)
- `backend/src/worker.ts` schedules 3 cron jobs but does NOT import or schedule `logOutcomes`
- Impact: `forecast_outcomes` table will not be populated automatically; `AccuracyDashboard` will stay in empty state until the CLI is run manually or a cron is added
- Mitigation: The CONTEXT.md and PROJECT.md both describe this as a CLI-first script ("Also schedulable as a cron"); the CLI is fully operational; the Phase 6 VERIFICATION.md marks VAL-01 SATISFIED
- Severity: **LOW** — functional gap in operational automation, not in the code itself; acceptable for v1.0 given the 7+ day data accumulation requirement before meaningful outcomes exist

### ⚠️ Orphaned Export (Tech Debt)

**`INCIDENT_TYPE_MAP` in `frontend/src/types/api.ts`**

- Exported constant mapping INRIX numeric incident type codes to string labels
- Neither `IncidentMarker.tsx` nor `IncidentPopup.tsx` imports it
- Incident type display uses the raw integer field; mapping happens inline where needed
- Severity: **LOW** — unused export, no runtime impact, no user-visible defect

## Known Operational Gaps (Not Code Defects)

These items require a live database or production environment to fully verify. They are documented in phase VERIFICATION.md files as human-only checks.

1. **Cron job deployment** — `worker.ts` and `ml/scripts/run_forecast.py` schedules must be registered on the deployment host manually
2. **TimescaleDB migration** — must be applied to a live PostgreSQL instance with TimescaleDB extension
3. **INRIX segment IDs** — must be populated from live `speed_readings` data (Phase 1 data accumulation required)
4. **Mapbox token** — `VITE_MAPBOX_TOKEN` must be set in production environment
5. **Sub-200ms API latency** — architecturally sound (pre-computed forecasts, cached endpoints) but cannot be confirmed without a running database under load
