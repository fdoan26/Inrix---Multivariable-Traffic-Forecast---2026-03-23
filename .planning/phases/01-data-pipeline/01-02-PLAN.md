<objective>
Build the three data collectors (INRIX speeds, INRIX incidents, Open-Meteo weather) and their database query modules. Each collector fetches data from an external API, validates with Zod, and stores in TimescaleDB.

Purpose: These are the core data ingestion functions that the worker process (Plan 03) will schedule on cron. Without working collectors, no data accumulates for the forecasting model.
Output: Three tested collector modules and three DB query modules, each independently callable.
</objective>

    4. Create `backend/src/collectors/inrix-speeds.ts`:
       - Import axios, InrixAuthService, checkBudget, recordCall, updateCallStatus, withRetry, SpeedResponseSchema, insertSpeedReadings, logJobStart, logJobEnd
       - Constants:
         - INRIX_SPEED_URL = 'https://segment-api.inrix.com/v1/segments/speed'
         - SF_BOX = '37.858|-122.541,37.699|-122.341'
       - Export async function `collectSpeeds(auth: InrixAuthService): Promise<{ segmentCount: number } | { skipped: true; reason: string }>`:
         - const jobId = await logJobStart('inrix_speeds')
         - const budget = await checkBudget()
         - If !budget.allowed: logJobEnd(jobId, 'skipped', 0, `Budget exhausted: ${budget.count} calls this week`), return { skipped: true, reason: 'budget_exhausted' }
         - const callId = await recordCall('inrix_speeds', INRIX_SPEED_URL)
         - const startTime = Date.now()
         - Wrap in withRetry (maxAttempts=3):
           - const token = await auth.getToken()
           - GET INRIX_SPEED_URL with params { box: SF_BOX, SpeedOutputFields: 'All', units: 0 }, headers { Authorization: `Bearer ${token}`, Accept: 'application/json' }, timeout: 30000
           - On 401 AxiosError: auth.invalidate() then rethrow (retry will get fresh token)
         - const parsed = SpeedResponseSchema.parse(response.data)
         - Flatten segments: parsed.result.segmentSpeed.flatMap(ss => ss.segments)
         - await insertSpeedReadings(segments, new Date())
         - await updateCallStatus(callId, 'success', 200, Date.now() - startTime)
         - await logJobEnd(jobId, 'success', segments.length)
         - return { segmentCount: segments.length }
         - On error: updateCallStatus(callId, 'error', ...), logJobEnd(jobId, 'error', 0, error.message), rethrow

       - Import { query } from '../connection'
       - Export interface WeatherRow { forecastHour: Date; temperatureC: number; precipitationMm: number; visibilityM: number; weatherCode: number; windSpeedKmh: number }
       - Export async function `upsertWeatherForecasts(rows: WeatherRow[]): Promise<number>`:
         - Build multi-row INSERT with ON CONFLICT:
           ```sql
           INSERT INTO weather_forecasts (forecast_hour, temperature_c, precipitation_mm, visibility_m, weather_code, wind_speed_kmh)
           VALUES ($1,$2,$3,$4,$5,$6), ...
           ON CONFLICT (forecast_hour) DO UPDATE SET
             fetched_at = NOW(),
             temperature_c = EXCLUDED.temperature_c,
             precipitation_mm = EXCLUDED.precipitation_mm,
             visibility_m = EXCLUDED.visibility_m,
             weather_code = EXCLUDED.weather_code,
             wind_speed_kmh = EXCLUDED.wind_speed_kmh
           ```
         - Return rowCount
