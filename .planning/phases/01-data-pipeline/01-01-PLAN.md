  <action>
    1. Create `backend/src/services/inrix-auth.ts`:
       - Import crypto from 'node:crypto' and axios
       - Export function `computeHashToken(appId: string, appKey: string): string` that:
         - Concatenates `${appId}|${appKey}`, lowercases, SHA1 hashes, returns hex string
       - Export class `InrixAuthService`:
         - Constructor takes `{ appId: string, appKey: string }`
         - Private fields: `cachedToken: string | null`, `tokenExpiry: Date | null`
         - Method `getToken(): Promise<string>`:
           - If cachedToken exists and tokenExpiry is more than 5 minutes away, return cachedToken
           - Otherwise: compute hashToken, GET `https://uas-api.inrix.com/v1/appToken?appId={appId}&hashToken={hashToken}`
           - Parse response: `response.data.result.token` and `response.data.result.expiry`
           - Cache both, return token
         - Method `invalidate(): void` -- clears cached token (for 401 retry)
       - TOKEN_URL = 'https://uas-api.inrix.com/v1/appToken'

    2. Create `backend/src/services/budget-tracker.ts`:
       - Import { query } from '../db/connection'
       - Import date-fns `startOfISOWeek` for week boundary calculation
       - Install date-fns: `npm install date-fns`
       - Export `WEEKLY_LIMIT` = parseInt(process.env.INRIX_BUDGET_WEEKLY_LIMIT || '1600')
       - Export async function `checkBudget(): Promise<{ allowed: boolean; count: number }>`:
         - Query: `SELECT COUNT(*)::int as count FROM api_call_log WHERE service LIKE 'inrix%' AND called_at >= $1`
         - Parameter: startOfISOWeek(new Date()).toISOString()
         - Return { allowed: count < WEEKLY_LIMIT, count }
       - Export async function `recordCall(service: string, endpoint: string): Promise<number>`:
         - INSERT INTO api_call_log (service, endpoint, status) VALUES ($1, $2, 'pending') RETURNING id
         - Return the id
       - Export async function `updateCallStatus(id: number, status: 'success' | 'error', statusCode?: number, responseTimeMs?: number, errorMessage?: string): Promise<void>`:
         - UPDATE api_call_log SET status=$1, status_code=$2, response_time_ms=$3, error_message=$4 WHERE id=$5

    3. Create `backend/src/services/retry.ts`:
       - Export interface RetryOptions { maxAttempts: number; baseDelayMs: number; maxDelayMs: number; onRetry?: (error: Error, attempt: number) => void }
       - Default options: maxAttempts=3, baseDelayMs=1000, maxDelayMs=30000
       - Export async function `withRetry<T>(fn: () => Promise<T>, options?: Partial<RetryOptions>): Promise<T>`:
         - Merge with defaults
         - Loop up to maxAttempts:
           - Try fn(), return on success
           - On failure: if last attempt, throw
           - Calculate delay: min(baseDelayMs * 2^attempt, maxDelayMs) + random jitter (0-500ms)
           - Call onRetry callback if provided
           - await new Promise(resolve => setTimeout(resolve, delay))
